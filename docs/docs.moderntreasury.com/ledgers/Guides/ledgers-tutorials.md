# Ledgers Quickstart

This series of tutorials explains how customers use Modern Treasury Ledgers according to their use case.

To try Ledgers, [sign up for an account](https://app.moderntreasury.com/sign_up?signupProduct=ledgers) and start using the API.

Ledgers is a specialty database designed to handle any money movement use case. Our customers use Ledgers to power:

* [Digital wallets](https://docs.moderntreasury.com/ledgers/docs/digital-wallet-tutorial#/) and in-network payments
* Programmatic payouts from [marketplaces](https://docs.moderntreasury.com/ledgers/docs/marketplace-tutorial#/)
* [Lending](https://docs.moderntreasury.com/ledgers/docs/lending-tutorial#/), including all its variants (BNPL, invoice factoring, mortgages, embedded lending, etc.)
* [Card](https://docs.moderntreasury.com/ledgers/docs/cards-tutorial#/) programs, issuing, and card payment authorization engines
* [Reward programs and points](https://docs.moderntreasury.com/ledgers/docs/points-rewards#/)
* Embedded payments, including [Bill Pay ](https://docs.moderntreasury.com/ledgers/docs/bill-payments-tutorial#/)
* Payroll and embedded [payroll](https://docs.moderntreasury.com/ledgers/docs/payroll#/)
* [Currency conversions and on/off ramps to crypto](https://docs.moderntreasury.com/ledgers/docs/fx-crypto#/)

The Quickstart tutorials here demonstrate common patterns for ledgering your application's money movements. Your application can write to the Ledgers API to record transactions, and read from the API for an immutable, real-time view of user balances.
