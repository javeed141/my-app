# Ledgers Overview

**Ledgers** is a fully managed cloud database for recording financial transactions and balances.

Any software product that has to track the movement of value to and from user accounts needs a reliable system of record. Having an immutable, scalable, double-entry ledger is the gold standard for businesses with complex or high-velocity payment flows.

You can try Ledgers by [signing up](https://app.moderntreasury.com/sign_up) for a sandbox account and making calls to the [Ledgers API](/platform/reference/ledger-object). Ledgers can be used independently of other functionality in Modern Treasury such as Payments. If you also use Payments, Modern Treasury provides powerful links for tying payments activity to the ledger.

The set of guides below will help you begin using Ledgers:

* Review our [use case](https://docs.moderntreasury.com/ledgers/docs/ledgers-tutorials) and [quickstart guides](https://docs.moderntreasury.com/ledgers/docs/ledgers-tutorials#/) for information on how to map Ledgers against your use case;
* Review our [guarantees](https://docs.moderntreasury.com/docs/ledgers-guarantees) to understand the basic principles governing debits and credits, write atomicity, immutability, and more;
* Review our guides to [ledger objects](https://docs.moderntreasury.com/docs/guide-to-ledger-objects); [debits and credits](https://docs.moderntreasury.com/docs/guide-to-debits-and-credits); and [ingesting data from other systems](https://docs.moderntreasury.com/docs/ingest-data-from-other-systems) for foundational concepts around object taxonomy, account normality and unifying data from other systems;
* Review [Link a Ledger Account](https://docs.moderntreasury.com/ledgers/docs/link-a-ledger-account-to-a-virtual-account) or [Link a Ledger Transaction](https://docs.moderntreasury.com/ledgers/docs/linking-ledger-transaction-to-payment-objects) to map the statuses of Ledger objects to other objects in the Modern Treasury product suite.
