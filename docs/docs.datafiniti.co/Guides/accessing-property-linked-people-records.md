# Accessing property data linked to people records

Connecting people to the properties they’re associated with can provide powerful insights. From targeting new homeowners and identifying real estate professionals to conducting risk assessments, this data plays a critical role. In this guide, we’ll leverage Datafiniti’s [People Data](https://docs.datafiniti.co/reference/people-data) search APIs to discover people linked to specific properties. Our aim is to help you enrich lead profiles, confirm ownership information, and maximize the value of your people data through integrating property data.

## Locating property records in people data

Whether you are looking for contact info of a property owner, to broker's information on buying / selling a property, Datafiniti's people data creates (when we can verify) a link from our people database to a property database. There are two places this can be found with our property data schema. You can discover these field more within our [People Data Schema](https://docs.datafiniti.co/docs/people-data-schema). Here are the response examples of the top level fields were they can be found:

`propertyKeys` - A unique key(s) generated by Datafiniti to link a person to our property database.

```json
{
    "propertyKeys": [
        "us/nc/brevard/7737greenvillehwy"
    ]
}
```

`propertiesOwned` - A unique key(s) generated by Datafiniti to link a homeowner to our property database.

```json
{
    "propertiesOwned": [
        "us/co/coloradosprings/4010baytowndrr"
    ]
}
```

<br />

`propertiesRepresented` - A unique key generated by Datafiniti to link brokers to our property database.

```json
{
    "propertiesRepresented": [
        "us/tx/kingwood/26870kingscrescentdr"
    ]
}
```

<br />

Once you have your key(s) for the person you wish to search the property data base for, you can use it to find the linked record in Datafiniti's property database. From detailed pricing history, to brokers associated with, transactional data, and more this will allow you greater insight into the person connect to the property.

# Accessing property data of an owner

We can use Datafiniti's [Property Data](https://docs.datafiniti.co/reference/properties) search endpoint to enhance the results found from a people search. This search method will provide more details about the specific homeowner's property. This guide will provide you with the steps to bridge the gap between property & people data.

## 1. Search for people that are homeowners

To start, we must first the homeowners that have been linked to our property data. This link can be identified with our `propertiesOwned` field. Using our [People Data](https://docs.datafiniti.co/reference/people-data) search endpoint first, we will filter for the `propertyKeys` field for any value with the following parameters:

* Lives within Denver, CO.
* Filter on any person that owns property.

```json
{
  "query": "country:US AND province:CO AND propertiesOwned:*",
  "num_records": 10
}
```

## 2. Finding the propertyOwned key

A `propertyOwned` key is generated per linked property Datafiniti identifies to the people record as the owner.\
You can find this key from the people data search API response:

```json

            "address": "4010 Baytown Dr",
            "city": "Colorado Springs",
            "country": "US",

            "dateUpdated": "2025-05-15T21:00:15Z",
            "domains": [
                "www.datafiniti.co"

            "firstName": "Stacy",
            "keys": [
                "stacy/stone/us/co/80916/4010baytowndr",

            "lastName": "Stone",

            "province": "CO",

                "us/co/coloradosprings/4010baytowndr"

```

We will save the `propertiesOwned` key from the response for searching the property database.

## 3. Lookup the property record with your propertiesOwned key

Now we will use the [Property Data](https://docs.datafiniti.co/docs/api-property-data) search endpoint to cross reference the people data's `propertyOwned` key to the property data's `people.people_key`. Using the `us/co/coloradosprings/2103greenwichcire` key from earlier we can directly filter the `people.people_key` field of our property data. You can use the following query to do so:

```json
{
  "query": "keys:\"us/co/coloradosprings/4010baytowndr\"",
  "num_records": 1
}
```

# Finding the brokers and their linked property records

We can use Datafiniti's [Property Data](https://docs.datafiniti.co/reference/property-data) search endpoint to enhance the results found from a people search. Providing more details about brokers contact info, currently listing, and more. This guide will provide you the steps to bridge the gap between property & people data.

## 1. Searching for brokers with linked property

To start, we must first find a broker that have been linked to our property data. This link for specifically brokers can be identified with our `propertiesRepresented` sub field. Using our [People Data](https://docs.datafiniti.co/reference/properties) search endpoint we are filter the property data's `brokers` field for `people_key`. You can use the following code to achieve this:

```json
{
	"query": "country:\"US\" AND province:TX AND city:kingwood AND propertiesRepresented:*",
	"num_records": 1
}
```

When linking property data to people records, especially in the context of brokers. Certain top-level fields in Datafiniti’s property data are particularly valuable.

* `geoLocation` - This can be used to generate a list of comparable properties around the area. For more information on how `geoLocation` filters work you can read about it [here](https://docs.datafiniti.co/docs/constructing-property-queries#geo-queries).
* `mostRecentStatus` - The current status of the property. This can be used to determine lead generate, interest in your service/goods, etc.
* `transactions` - The recorded sales transaction of a property.
