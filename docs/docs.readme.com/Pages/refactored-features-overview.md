# What's New with ReadMe Refactored

ReadMe Refactored represents the future of ReadMe, and includes improvements to almost every aspect of the product, plus powerful collaboration and AI features that are exclusive to this new experience!

### Built In AI Features to Empower Your Team and End Devs!

Whether you’d like help writing your docs, help your developers understand your API faster, or get useful suggestions about how to improve your docs, it’s all possible in ReadMe with our <Anchor label="enhanced AI tools" target="_blank" href="https://docs.readme.com/main/docs/ai-overview">enhanced AI tools</Anchor>, including our built-in linter, docs audit, writing assistant, and more.

### New Editing UI

All docs editing now happens on your live hub, allowing you to make edits directly in your docs and easily preview content and appearance changes before they go live.

<Image align="center" border={false} src="https://files.readme.io/05a676c4f26ee484419da1748b7537f4c36b4e64d7ccfe49af7c9a96deb3a09a-july_2025_editing_ui.gif" />

### Global Custom MDX Components

Add dynamic and interactive content like live code samples and data visualizations to your docs <Anchor label="thanks to MDX" target="_blank" href="https://docs.readme.com/main/docs/mdx#/">thanks to MDX</Anchor>! Choose from pre-built components, <Anchor label="explore our GitHub marketplace" target="_blank" href="https://github.com/readmeio/marketplace/">explore our GitHub marketplace</Anchor>, or <Anchor label="build your own custom components " target="_blank" href="https://docs.readme.com/main/docs/building-custom-mdx-components#/">build your own custom components </Anchor> using  a combination of React, JSX, and Tailwind styling and easily reuse them throughout your different child projects.

<Image align="center" border={false} src="https://files.readme.io/dcdb8875413f272d7b9cf27a12234b9ec724e8e07990659099f62b0e54a60f2a-july_2025_mdx_components.gif" />

### Bi-Directional Syncing with GitHub

Now different members of your team can work wherever they’re most comfortable— whether in their local dev environment or directly in ReadMe’s editor— and changes will <Anchor label="sync seamlessly between both" target="_blank" href="https://docs.readme.com/main/docs/bi-directional-sync#/">sync seamlessly between both</Anchor>, no manual updates necessary. As an Enterprise customer, you can also enable a GitHub Enterprise Server for your Group, in addition to GitHub Enterprise Cloud.

<Image align="center" border={false} src="https://files.readme.io/0db843f39eaab6ab1e0f8d5a06c37e7e7ee686142fd965a2e79c3083c6835965-july_2025_bi-di_visual.png" />

### Branches, Reviews, and Editor Role

Now you can create branches in your docs, just like in your code. <Anchor label="Branches" target="_blank" href="https://docs.readme.com/main/docs/branches#/">Branches</Anchor> allow your team to make changes and review them in a preview environment, plus send changes to teammates for review. No more one-size-fits-all staging—create as many branches as you need and merge only what’s ready.

<Image align="center" border={false} src="https://files.readme.io/ae5da99db1f71fddc03e3f2d41ec4da271e7efa5c300e788111b92591584fb3d-july_2025_branches.gif" />

With Branch Reviews, you're able to compare changes between the branched version and your main version before you merge. Branch Reviews provide a simple, structured way to see exactly what's changed on a branch, review in context, and approve edits before they go live.

<Image align="center" border={false} src="https://files.readme.io/aac34170b5a5642e97460bddb20e4c0a4c3cf26ef742bd1721c6f6080f26e50b-Aug_2025_branch_reviews.gif" />

<Callout icon="👍" theme="okay">
  Branches and Reviews are available on Business and Enterprise plans only.
</Callout>
